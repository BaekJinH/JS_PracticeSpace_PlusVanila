General Formatting Rules

  • 모든 속성은 영문 소문자로만 작성
  • 공백이 포함된 폰트명, 한글 폰트명, 문자열 데이터 타입, filter 속성의 파라미터값은 작은따옴표(' ')로 감싸며, @charset 선언 시에는 속성 값을 큰따옴표(" ")로 감싼다. 그 외의 경우에는 따옴표를 사용하지 않음
  • 마지막 속성의 세미콜론(;)은 삭제
  • CSS 코드를 작성할 때는 들여쓰기를 하지 않는다. 단, 중괄호”{}”가 중첩되는 경우 예외
  • 들여쓰기는 하드탭으로 변경 1탭 4스페이스로 적용은 탭으로 적용
  • 각각 속성 선언 콜론(:)뒤에는 공백, 쉼표로 구분되는 선택자 간 공백, 속성 간 공백 및 속성 값 사이 공백, 중괄호 좌, 우 공백은 제거
  • 속성값들과 색상값들에 소숫점이 필요할 경우 0.를 사용하지 않음 (예: .5 또는 -.5px)
  • 16진수 값들은 소문자로 표현하며 축약형태로 사용
  • 0값들에서는 단위생략. (예: margin:0;)
  • 폰트의 기본 단위는 px을 사용하며, %, em은 선택적으로 사용가능

code pattern

                        .selector[type=text],.selector-secondary{margin-bottom:15px;padding:15px;background-color:rgba(0,0,0,.5);box-shadow:0 1px 2px #ccc,inset 0 1px 0 #fff}

속성 선언 순서

  • 속성에 선언순서는 포지션 관련 > 박스 모델 > 시각 효과 > 타이포그라피 > 기타
포지션 관련 선언들이 가장 먼저 옵니다. 문서의 정상적인 흐름에서 엘리먼트들을 없앨 수 있고, 박스 모델 관련 스타일들을 재정의할 수 있기 때문입니다.
박스 모델이 그 다음으로 오는데, 이것은 해당 엘리먼트의 크기와 위치를 지정하기 때문입니다.
나머지는 이 앞의 두 섹션에 영향을 주지 않거나 해당 엘리먼트의 내부에서만 일어나기 때문에 나중에 배치 시킵니다.

code pattern

                        .declaration-order{
                        /* Positioning */
                        position:absolute;top:0;right:0;bottom:0;left:0;

                        /* Box-model */
                        display:block;float:right;width:100px;height:100px;margin:10px;padding:20px;

                        /* Visual */
                        border:1px solid #e5e5e5;border-radius:3px;background-color:#f5f5f5;
                                                
                        /* Typography */
                        color:#333;font:normal 14px artial,sans-serif;text-align:center;line-height:1.3;
                        
                        /* Misc */
                        opacity:1;z-index:100

@import 사용 금지

<link>에 비해 @import는 상대적으로 느리고 추가 페이지 요청을 필요로 하기 때문에 로딩관련하여 예측할 수 없는 문제를 야기할 수 경우가 있어 @import의 사용을 피하고 <link>를 사용

code pattern

                        
                        <link rel="stylesheet" href="/contents.css">

Class Naming

  • 클래스 이름은 소문자와 하이픈(-)을 사용하며, 하이픈은 관련 클래스에서 자연스러운 단어별브레이크를 제공(editplus의 경우 js지원, 서브라임 텍스트는 모든형식지원)
  • 의미없는 클래스명 보다는 명시적으로 선언
  • 클래스 이름은 축약형태로 선언하며, 과도한 축약은 피한다(ex. .a)
  • 의미 있는 이름을 사용하여 표현보다는 구조라든가 목적을 나타내는 네이밍으로 표현
  • prefix는 베이스나 가장 가까운 부모를 바탕으로 네이밍

구성

  • 콤포넌트 및 모듈 그룹에 맞춰 코드를 구성
  • 일관적인 상하관계에 맞추어 개발
  • 페이지 대신 콤포넌트 및 모듈별로 그룹핑

에디터 설정

  • 파일 저장시 utf-8 인코딩으로 저장
  • 하드탭 설정을 공백 문자 네개로 구성된 1Tab으로 세팅
