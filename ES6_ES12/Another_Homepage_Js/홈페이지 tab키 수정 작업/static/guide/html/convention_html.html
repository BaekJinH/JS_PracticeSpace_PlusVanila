K-DATA Guide

General Formatting Rules

  • 들여쓰기는 하드탭으로 변경 1탭 4스페이스로 적용은 탭으로 적용
  • 속성값을 지정할 때에는 작은 따옴표 대신 항상 큰 따옴표를 사용
  • 단독 태그 엘리먼트들는 닫는태그를 생략 – HTML5 스펙에서는 선택 사항
  • 소스상의 tag, 속성(&values), css셀렉터(&property)등은 소문자만 사용(문자열 제외)
  • html5에서의 숫자의 무조건인 링크는 메타 포맷으로 방지(계좌번호, 전화번호등)

code pattern

                            <!DOCTYPE html>
                            <html lang="ko">
                            <head>
                            <meta charset="UTF-8">
                            <meta http-equiv="X-UA-Compatible" content="ie=edge">
							<meta name="title" content="">
							<meta name="description" content="">
							<meta name="keywords" content="">
							<meta name="format-detection" content="telephone=no">
                            <title>Page title</title>
                            </head>
                            <body>
                                <img src="/logo.png" alt="K-DATA 로고">
                                
K-DATA

                            </body>
                            </html>

HTML5 DOCTYPE

  • 문서형식 선언을 HTML5로 지정함(htmlShiv사용하지 않음)
  • 문서 타입을 선언함으로써, 표준 모드로 렌더링되어 브라우저에서 일관적으로 렌더링을 보장

code pattern

                        <!DOCTYPE html>

언어 속성

HTML5 스펙의 정의
작성자는 문서에 쓰인 언어를 알리기 위해 lang 속성을 루트의 html 엘리먼트에 명시하는 것이 좋다. 이것은 텍스트 리더기와 같은 도구들이 어떤 언어와 발음를 써야 하는지, 어떤 번역 도구를 써야 하는지 등에 대해 내부적으로 결정할 수 있게 도와준다.

code pattern

                        <html lang="en">
                        <html lang="ko">

Encoding

  • 문자열의 명시적인 선언으로 컨텐츠의 빠르고 렌더링하는 목적으로 상단에 선언

code pattern

                        <meta charset="UTF-8">

인터넷 익스플로러 호환 모드

  • 인터넷 익스플로러는 태그를 통해 어떤 버전의 인터넷 익스플로러를 사용해야 하는지를 명시함으로써 문서의 호환성을 지원합니다. 딱히 특별한 상황이 아니라면 엣지 모드를 사용하여 인터넷 익스플로러에게 가장 최신 버전을 사용하라고 하면 됩니다.

code pattern

                        <meta http-equiv="X-UA-Compatible" content="ie=edge">

Docu. Title

  • 동일한 규칙으로 문서의 제목을 일원화하여 SEO최적화
  • 문서의 타이틀태그에 컨텐츠의 제목을 포함시켜 검색서비스 노출에 유리
  • 타이틀에 사용된 특수문자는 기호가 아닌 Entity코드(&lt;, &ndash; &verbar;)를 사용
crawling : 많은 컴퓨터에 분산 저장되어 있는 문서를 수집하여 검색대상의 색인으로 포함시키는 기술

code pattern

                        <title>활용가이드 – 메뉴얼 | K-DATA</title>

External Link

HTML5 스펙에 따르면 css와 js파일을 불러올 때 type 속성을 생략가능 text/css 와 text/javascript 속성값이 각각 기본값(선택적용)

code pattern

                    
                    <link rel="stylesheet" href="common.css">
                    
                    <script src="common.js">

Boolean Attr.

불리언 속성값은 생략가능 XHTML에서는 속성값 지정이 필요했지만 HTML5에서는 그런 제약이 없음(선택적용)

불리언 태그 속성이 태그 안에 존재한다는 것 자체가 true를 의미한다.
반대로 불리언 태그 속성이 태그 안에 선언되지 않았다면 그것은 false를 의미한다.

code pattern

                        <input type="text" disabled>
                        <input type="checkbox" checked>
                        <select>
                        <option value="1" selected>
                        </select>

Entity Code

모든 문자를 entity로 변환할 필요는 없으며, 아래의 케이스만 적용

code pattern

                        
                        &middot;
                        
                        &lt;
                        
                        &gt;
                        
                        &amp;
